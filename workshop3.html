Why Python

You've got a shell and a repo. Now the language that turns them into science.

Python is…

High-levelyou don't manage memory — you describe the idea
Interpretedruns without compiling; you can poke at it line by line
Readableclear, minimal syntax — good Python often reads like pseudocode
Cross-platformthe same code runs on your laptop and on Fiji

Why scientists use it

Large ecosystemNumPy, pandas, SciPy, BioPython, scikit-learn — someone has already written the hard part
Fast prototypingtry an idea before lunch, iterate after
Communityvast user base, excellent docs, an answered Stack Overflow question for nearly everything
Glue languagebridges C++, Java, R, and bash — it holds pipelines together

Speak Python

Third language, same lesson. Bash gave you imperative verbs; git gave you two-word verbs. Python gives you full sentences — with a subject, a verb, and a rule that whitespace is grammar. Tap any word to hear it in English.

verb / functionkeywordvalue name (variable)grammar particles

counts[nuc] += 1

return peptide

The little words — Python's particles

= vs ==“let this name mean…” vs “are these equal?” — the most common beginner bug in any language
: + indentthe colon opens a block; the indentation is the block. Python has no braces — whitespace is syntax
#“ignore the rest of this line” — a note to humans
[a:b]a slice: from a, up to (not including) b. seq[:5] first five · seq[-1] last one · seq[::-1] reversed
f'…{x}…'an f-string: drop the value of x right into the text
.method()“ask this object to do something to itself” — line.strip(), dna.replace('T','U')
The [start:end:step] slice is the sentence you'll use most today. sequence[::-1] reads “everything, backwards” — half of a reverse complement, in nine characters.

rev_comp = ''.join(complement[nuc] for nuc in sequence[::-1])

Step 5. Write translate(dna): transcribe to RNA, then read codon by codon into a peptide, stopping at the first stop codon. Prove it works on a tiny sequence, then point it at a real region of the genome.

hint

Transcription is one call: dna.replace('T','U'). Then walk the RNA three letters at a time — range(0, len(rna)-2, 3) — looking each codon up in codon_map and breaking on 'Stop'. Test it on 'ATGTCGTAA': you should get 'MS'. Genes start at an ATG; sequence.find('ATG') finds the first one.

solution
def translate(dna):
    rna = dna.replace('T', 'U')
    peptide = ''
    for i in range(0, len(rna) - 2, 3):
        amino_acid = codon_map[rna[i:i+3]]
        if amino_acid == 'Stop':
            break
        peptide += amino_acid
    return peptide

assert translate('ATGTCGTAA') == 'MS'      # prove it before you trust it

start = sequence.find('ATG')               # first start codon in the genome
print(f'first ATG at index {start}')
print(translate(sequence[start:]))
the classic bioinformatics bug 🐛

Real gene coordinates come from a genome's annotation (GenBank lists lambda's genes on the J02459.1 record). Watch out: GenBank counts from 1 and includes both endpoints; Python counts from 0 and excludes the end. A gene annotated 191..736 is sequence[190:736] in Python. Off-by-one here silently frameshifts your protein into gibberish — check that your slice actually starts with ATG before you believe the output.

C1 · Reverse translation. How many distinct mRNA sequences — modulo 1,000,000 — could have produced the given peptide?

hint

This is a counting problem, not a search. Each amino acid is coded by some number of codons; multiply those counts together. The number explodes fast, so take % 1000000 as you go, not at the end. collections.Counter(codon_map.values()) gives you the degeneracy of each amino acid for free.

solution
from collections import Counter
codons_per_aa = Counter(codon_map.values())

count = 1
for aa in peptides:
    count = (count * codons_per_aa[aa]) % 1000000

print(f'{count:,}')

the classic Rosalind version also counts the stop codon — multiply once more by 3

C2 · Counting k-mers. Write count_kmers(sequence, k) returning how often each length-k substring occurs. Then plot the 10 most common 8-mers.

hint

Careful with the loop bound: stop at len(sequence) - k + 1, or the last few "k-mers" come out shorter than k. dict.get(key, 0) saves you an if key not in check. Sort with sorted(d.items(), key=lambda x: x[1], reverse=True)[:10].

solution
def count_kmers(sequence, k):
    kmers = {}
    for i in range(len(sequence) - k + 1):   # every k-mer full length
        kmer = sequence[i:i+k]
        kmers[kmer] = kmers.get(kmer, 0) + 1
    return kmers

top = dict(sorted(count_kmers(sequence, 8).items(),
                  key=lambda x: x[1], reverse=True)[:10])

C3 · GC content windows. Write gc_window(sequence, w) returning GC% in each sliding window, then lineplot it across the whole genome. What happens as w grows — and why?

hint

Naively recounting each window is O(n·w) and slow on 48 kb. Instead slide: count the first window, then for each step subtract the base leaving and add the base entering. Self-check: gc_window('ATGTCGTAAT', 4) must give [25.0, 50.0, 75.0, 50.0, 50.0, 25.0, 0.0].

solution
def gc_window(sequence, w):
    counts = {nuc: 0 for nuc in ['A', 'T', 'G', 'C']}
    for nuc in sequence[:w]:
        counts[nuc] += 1

    gc = [(counts['G'] + counts['C']) / w * 100]
    for i in range(len(sequence) - w):
        counts[sequence[i]] -= 1       # base leaving the window
        counts[sequence[i+w]] += 1     # base entering it
        gc.append((counts['G'] + counts['C']) / w * 100)
    return gc

bigger w smooths the curve — local variation averages toward the genome mean (~49.9%)
