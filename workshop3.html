Why Python

You've got a shell and a repo. Now the language that turns them into science.

Python is…

High-levelyou don't manage memory — you describe the idea
Interpretedruns without compiling; you can poke at it line by line
Readableclear, minimal syntax — good Python often reads like pseudocode
Cross-platformthe same code runs on your laptop and on Fiji

Why scientists use it

Large ecosystemNumPy, pandas, SciPy, BioPython, scikit-learn — someone has already written the hard part
Fast prototypingtry an idea before lunch, iterate after
Communityvast user base, excellent docs, an answered Stack Overflow question for nearly everything
Glue languagebridges C++, Java, R, and bash — it holds pipelines together
Today you're handed a real knockout experiment: an RNA-binding protein called ZFP36L2 — which normally grabs certain mRNAs and marks them for destruction — deleted across six mouse tissues. Speak Python

Third language, same lesson. Bash gave you imperative verbs; git gave you two-word verbs. Python gives you full sentences — with a subject, a verb, and a rule that whitespace is grammar. Tap any word to hear it in English.

verb / functionkeywordvalue name (variable)grammar particles
universal = lung_up & liver_up & bm_up

for gene in up_genes:
n_tissues[gene] += 1

def up_regulated(table):
return genes

The little words — Python's particles

= vs ==“let this name mean…” vs “are these equal?” — the most common beginner bug in any language
: + indentthe colon opens a block; the indentation is the block. Python has no braces — whitespace is syntax
& · |set math: a & b = in both (intersection) · a | b = in either (union). Today's whole hunt is one long &
#“ignore the rest of this line” — a note to humans
f'…{x}…'an f-string: drop the value of x right into the text — f'{gene}: up in {n} tissues'
.method()“ask this object to do something to itself” — line.strip(), row.split(','), genes.add(name)
The one to burn in today is &. Six tissues, five ampersands: lung & liver & bm & spleen & ovary & kidney — and whatever survives is what ZFP36L2 never lets go of, anywhere.

      C1The near-universal targets
      
      
      
    

    
    

    

    
hint
Load each tissue's up-set (set(open(t+'_up.txt').read().split())).
      Count appearances: Counter(g for s in sets for g in s), then keep genes whose count is >= 4.
      To see names: read multi_tissue_up_symbols.csv into an id→symbol dict.

    
what you'll find 🧬
A tight cluster of immune genes — MHC class I (H2-Q5/6/7),
      granzyme B (Gzmb), Clec4n, serpins, interferon-induced Gvin/Tgtp. ZFP36L2's fingerprints are
      all over the immune system — which is exactly the paper's story.

  


  

    

      C2Does it bind them directly?
      
      
      
    

    
    

    

    
hint
Union the six up-sets with | to get "up in any tissue".
      Load the eCLIP genes: set(open('eclip_3utr_genes.txt').read().split()). The direct targets are the
      intersection (&) of the two — genes that are both up-regulated and bound.

    
the point 🔬
Two independent experiments — a knockout (what changes) and eCLIP
      (what's bound) — pointing at the same genes is far stronger evidence than either alone. That's how you build a
      real argument in a paper: converging lines of evidence.

  


  

    

      C3The ARE motif
      
      
      
    

    
    

    

    
hint
Slide a 5-wide window: sum(1 for i in range(len(seq)-4) if seq[i:i+5]=='AUUUA').
      Overlapping counts are fine — 'AUUUAUUUA' has two.

    
going further 🔬
The paper scores whole 3′UTRs with AREScore (proximity and
      density of AU-rich pentamers) and finds up-regulated genes score higher (mean 3.75 vs 2.82). Pull a few
      3′UTR sequences from Ensembl (see data/zfp36l2/fetch_reference.sh) and build your own mini
      AREScore — that's a genuine research-grade extension.
