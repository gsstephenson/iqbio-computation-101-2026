Follow-along manual

Setup Guides — step by step, no experience assumed

Interactive how-tos for Homework Zero and the first workshops. Pick your operating system once and every guide adapts. The dark windows show exactly what your terminal will look like — commands have copy buttons, expected output is shown underneath, and every guide ends with a fix-it box for when things break.

Guide 1 · Connect to Fiji

Homework Zero, item 4 → your first login in Workshop 1.

STEP 1 Open a terminal ▶

Press ⌘ + Space, type Terminal, hit Enter. That black window is your new best friend.

Open the Start menu, type PowerShell, hit Enter. (Windows 10/11 ships with an SSH client built in — you do not need WSL or PuTTY.)

You likely know this already — Ctrl + Alt + T on most distributions.

STEP 2 Check your SSH client ▶

Type this and press Enter (this is your Homework Zero screenshot):

$ ssh -V

OpenSSH_9.6p1, LibreSSL 3.3.6 ← any version is fine

Preinstalled on every Mac — this will just work.
"ssh is not recognized"? Settings → Apps → Optional Features → Add a feature → install OpenSSH Client, then reopen PowerShell.
If missing: sudo apt install openssh-client (or your distro's equivalent).
STEP 3 Off campus? Connect the VPN first ▶

Off campus, CU servers are simply unreachable without the VPN — connection timed out is what that looks like. Install Cisco Secure Client (formerly AnyConnect) using OIT's install guide, connect to vpn.colorado.edu (IdentiKey + Duo), then SSH normally. On campus wifi, skip this step.

STEP 4 Log in ▶

Replace identikey with your actual IdentiKey username:

$ ssh identikey@fiji.colorado.edu

The authenticity of host 'fiji.colorado.edu' can't be established. ED25519 key fingerprint is SHA256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx. Are you sure you want to continue connecting (yes/no/[fingerprint])? ← type: yes identikey@fiji.colorado.edu's password: ← typing is INVISIBLE — that's normal, just type and hit Enter

First time connecting, the "authenticity" warning is expected — it's your computer meeting Fiji for the first time. Type yes.
STEP 5 Look around — you're on a supercomputer ▶ 
[you@fiji ~]$ hostname

fiji.colorado.edu

[you@fiji ~]$ pwd

/home/identikey ← your home directory; where scratch & projects live comes in Workshop 1

Before you leave, grab your proof line for the readiness check — run this on Fiji and copy what it prints:

[you@fiji ~]$ echo "IQBIO-PROOF $(whoami)@$(hostname) $(date -u +%Y-%m-%dT%H:%M)Z"

IQBIO-PROOF identikey@fiji.colorado.edu 2026-08-10T21:04Z ← paste this into Verify & Submit

Done exploring? exit logs you out.

Fix-it box: Connection timed out → you're off-campus without VPN, or on guest wifi. Permission denied → IdentiKey typo or wrong password. Still stuck → screenshot the error, bring it to Workshop 1 — that's what it's for.

Guide 2 · SSH key → GitHub, on Fiji

The heart of Workshop 2 — done live, together, so nobody leaves broken.

STEP 1 Generate your keypair (on Fiji) ▶

Logged into Fiji, run (use the email on your GitHub account):

[you@fiji ~]$ ssh-keygen -t ed25519 -C "your_email@colorado.edu"

Enter file in which to save the key (/home/you/.ssh/id_ed25519): ← just press Enter Enter passphrase (empty for no passphrase): ← Enter twice for none (fine for today)

A keypair is two files: id_ed25519 (private — never leaves Fiji, never gets shared) and id_ed25519.pub (public — this one goes to GitHub).

STEP 2 Show & copy your public key ▶ 
[you@fiji ~]$ cat ~/.ssh/id_ed25519.pub

ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAII... your_email@colorado.edu

Select the entire output line in your terminal and copy it — every character, no extras.

STEP 3 Add it to GitHub ▶

On GitHub: click your avatar → Settings → SSH and GPG keys → New SSH key. You'll see this form:

🔒 github.com/settings/ssh/new
Title fiji Key type Authentication Key Key ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAII... your_email@colorado.edu

Simulated for clarity — the real page looks like this. Title it "fiji" so future-you knows which machine this key belongs to.

STEP 4 The moment of truth ▶ 
[you@fiji ~]$ ssh -T git@github.com

Hi YOUR-USERNAME! You've successfully authenticated, but GitHub does not provide shell access.

If GitHub greets you by name — you're wired up. This is the step that used to cost new students weeks.

Fix-it box: Permission denied (publickey) → the key on GitHub isn't the full line (re-copy from cat, watch for line breaks), or you skipped Step 3. First connection also asks the yes/no authenticity question — type yes.

Guide 3 · Your first repo — the full loop

Workshop 3's capstone: everything from Workshops 1 & 2, combined.

STEP 1 Tell git who you are (once, ever) ▶ 
[you@fiji ~]$ git config --global user.name "Your Name"
[you@fiji ~]$ git config --global user.email "your_email@colorado.edu"
[you@fiji ~]$ git config --global init.defaultBranch main

Use the same email as your GitHub account so your commits count as yours. The last line makes every new repo start on branch main — matching GitHub and the outputs below.

STEP 2 Make a project and put it under version control ▶ 
[you@fiji ~]$ mkdir my-first-analysis && cd my-first-analysis && git init

Initialized empty Git repository in /home/you/my-first-analysis/.git/

[you@fiji my-first-analysis]$ echo "# My First Analysis" > README.md
STEP 3 First commit ▶ 
[you@fiji my-first-analysis]$ git add README.md && git commit -m "First commit"

[main (root-commit) a1b2c3d] First commit 1 file changed, 1 insertion(+)

STEP 4 Create the empty repo on GitHub ▶

On GitHub: + (top right) → New repository:

🔒 github.com/new
Repository name my-first-analysis Visibility ● Public  ○ Private

Leave "Add a README" UNCHECKED — your repo already has one. An empty GitHub repo avoids a first-push conflict.

STEP 5 Link and push ▶

Swap in your GitHub username:

[you@fiji my-first-analysis]$ git branch -M main
[you@fiji my-first-analysis]$ git remote add origin git@github.com:USERNAME/my-first-analysis.git
[you@fiji my-first-analysis]$ git push -u origin main

Writing objects: 100% (3/3), done. branch 'main' set up to track 'origin/main'.

Refresh the GitHub page — your README is live. That's code written on a supercomputer, published to the world.

STEP 6 Close the loop ▶

On your laptop (with its own key added to GitHub — same as Guide 2):

$ git clone git@github.com:USERNAME/my-first-analysis.git

Cloning into 'my-first-analysis'... done.

Edit on the laptop, push; pull on Fiji. Same project, every machine, always in sync — this loop is the daily rhythm of research computing.

Fix-it box: remote origin already exists → git remote remove origin and re-add. Permission denied on push → Guide 2's test (ssh -T git@github.com) must greet you by name first.
