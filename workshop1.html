02

The command line

Linux powers nearly every server, cloud, and supercomputer on Earth — Fiji included. The command line is how you talk to it: faster than clicking for big data, scriptable, reproducible, and controllable from anywhere over SSH. Nobody memorizes all of it — getting good at looking things up is the skill.

Speak like a computer — the grammar of the shell

The command line isn't magic words — it's a language, and every command is an imperative sentence: you're giving an order. The command is the verb, flags are adverbs (how to do it), and arguments are the objects (what to do it to). Tap any word below to hear it in English.

verbadverb (flag)object person & machinegrammar particles
ls -lh /home/

ssh sam@fiji.colorado.edu

scp -r results/ sam@fiji.colorado.edu:/scratch/Users/sam/

The little words — a pocket dictionary of paths

WordMeans
.“here” — the directory I'm standing in (./script.sh = “run the one right here”)
..“up one” — my parent directory
~“my home”
/at the start: “the root of the filesystem” · in the middle: just a separator
-“where I just was” (cd - bounces you back)
*“anything” — *.txt = every file ending in .txt
Once you can parse, you never memorize again — you read. And when a sentence stumps you: almost every command answers --help (or -h), and man command opens its full dictionary entry — q to quit.

Combining commands — where it gets powerful

$ ls -l | grep ".txt"   # | pipes output of one command into the next
$ ls -l > listing.txt    # > sends output to a file (>> appends)
$ cd data && ls -lh      # && chains commands: second runs if first succeeds
This shell is bash — a full scripting language with variables, loops, and functions. Workshop exercises build up to writing your first script.

Editing files in the terminal

$ nano notes.txt   # simple: Ctrl+O save · Ctrl+X exit

vim exists too — powerful, steeper curve. Trapped inside it? Esc then :q! — everyone's been there.

Fiji — your supercomputer

Fiji is CU Boulder's BioFrontiers high-performance computing (HPC) cluster — bioinformatics, genomics, ML, simulations. Free for first-year IQ Biology students, managed by BioFrontiers IT (BIT). Jes covered the big picture at 2:00; here's what you need at your fingertips.

Where files live — and why it matters

PathWhat it's for
~/ (home)Permanent, small — code, configs, notes. Not big data.
/scratch/Users/<identikey>Temporary, high-speed I/O — where jobs read/write big data. Not backed up.
/scratch/Shares/Temporary group space (request access)
/Shares/ · /projects/Shared group space (request access)
 
[you@fiji ~]$ du -sh *     # how big is everything here?
[you@fiji ~]$ df -h .      # how full is this filesystem?

Get the data onto Fiji

Pull the ZFP36L2 dataset you'll use all week with wget, then unpack it with tar — two commands you'll reach for constantly.

[you@fiji ~]$ wget https://github.com/gsstephenson/iqbio-computation-101-2026/raw/main/data/zfp36l2-data.tar.gz
[you@fiji ~]$ tar -xzf zfp36l2-data.tar.gz        # unpacks a zfp36l2/ folder
[you@fiji ~]$ ls zfp36l2/de                        # the six tissue DE tables

BM_DE.csv Kidney_DE.csv Liver_DE.csv Lung_DE.csv Ovary_DE.csv Spleen_DE.csv

This is the exact data your Workshop 3 analysis loads — you're meeting it a day early, on the cluster.

The one etiquette rule

Never run heavy computation on the login node. The login node is the shared front door — everyone's on it. Real work goes through Slurm (next section), which hands you a compute node. Break this rule and BIT will email you; everyone gets that email once.

Software — modules & conda

[you@fiji ~]$ module avail          # what's installed?
[you@fiji ~]$ module load python/3.11.3   # newest Python 3 on Fiji (2.7–3.11 available)
[you@fiji ~]$ module list           # what's loaded now?
Custom Python/R environments → conda. If conda misbehaves after a module load, run eval "$(conda shell.bash hook)" first (thanks, Ben Braun).

Moving files on and off Fiji

Run these from your laptop's terminal, not from Fiji:

$ scp identikey@fiji.colorado.edu:~/results.csv .     # download
$ scp data.csv identikey@fiji.colorado.edu:~/         # upload
$ rsync -avhP identikey@fiji.colorado.edu:~/results/ ./results/

rsync > scp for big or repeated transfers — resumable, shows progress, skips unchanged files

04

Slurm — asking for compute

Slurm — Fiji's workload manager (the name began as Simple Linux Utility for Resource Management) — is the traffic controller: it queues jobs, allocates CPUs/memory/time, and balances load across compute nodes. You describe what you need; Slurm finds you a machine.

Three ways to run

TypeCommandWhen
Batchsbatch job.shThe workhorse — submit a script, walk away
Interactivesrun / sallocDebugging, quick tests — srun --pty bash drops you straight onto a compute node; salloc reserves nodes you then attach to
Monitoringsqueue, sacctWatching what's running / what ran
Partitions (add --partition=NAME): short is the default (jobs up to 1 day) · long up to 30 days · highmem for big-memory jobs · nvidia-a100 / titan for GPUs. See live capacity with sinfo -s.

Build your first job script — live

Fill in the fields; the script writes itself. Copy it into nano job.sh on Fiji, then sbatch job.sh.

[you@fiji ~]$ squeue --me            # my running/queued jobs

I2. Print just the first three gene rows of data/bm_de.csv — skip the header — using a pipe.

I3. The practice file data/genes.txt lists a gene once per peak found. Which genes show up most often, most frequent first?

      
hint
uniq -c only counts adjacent duplicates — sort first. Then sort the counts numerically, reversed.

      
solution
$ sort data/genes.txt | uniq -c | sort -rn
      3 Ccl8
      2 Gzmb
      1 Clec7a

    
I4. Pull out just the gene symbols (the last column) from data/bm_de.csv.

      
hint
cut with a delimiter flag and a field number — the CSV is comma-separated, the symbol is field 4.

      
solution
$ cut -d',' -f4 data/bm_de.csv

    
I5. How many peaks mention Ccl8 in data/genes.txt — regardless of case? One pipeline, one number out.

      
hint
grep has a case-insensitive flag and a count flag — you may not even need the pipe.

      
solution
$ grep -ic "ccl8" data/genes.txt   # or: grep -i ccl8 data/genes.txt | wc -l → 3

E1. Real parallelism: an array job with one task per tissue. Task N picks its tissue, counts the genes in that tissue's DE table, and writes the count to results/.

      
hint
#SBATCH --array=1-6; index a bash array with $SLURM_ARRAY_TASK_ID (arrays are 0-based, so subtract 1).

      
solution
#!/bin/bash
#SBATCH --array=1-6
#SBATCH --time=00:05:00
#SBATCH --mem=1G
tissues=(Lung Liver BM Spleen Ovary Kidney)
t=${tissues[$SLURM_ARRAY_TASK_ID-1]}
echo "$t $(wc -l < zfp36l2/de/${t}_DE.csv)" > results/count_$t.txt
$ mkdir -p results && sbatch array.sh && squeue --me   # six tasks, one submission

hint

solution
$ rsync -avhP identikey@fiji.colorado.edu:~/results/ ./results/
$ rsync -avhP ./results/count_BM.txt identikey@fiji.colorado.edu:~/results/

E4. One pipeline, real question: which tissue has the most up-regulated genes? Each zfp36l2/derived/<Tissue>_up_geneids.txt holds one tissue's up-gene IDs — rank the tissues by how many.

      
hint

solution
$ wc -l zfp36l2/derived/*_up_geneids.txt | sort -rn | head
# bone marrow tops the list — the tissue ZFP36L2 loss hits hardest

hint

"Anything my laptop sends to localhost:8888 is tunneled through SSH to port 8888 on Fiji" — i.e., Fiji's Jupyter appears in your local browser.
